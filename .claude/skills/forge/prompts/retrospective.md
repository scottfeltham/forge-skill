# Retrospective Prompts

Use these questions after completing a cycle to capture learnings.

## What Went Well

- What worked better than expected?
- What would you do the same way again?
- What tools or techniques were particularly effective?
- Where did the FORGE process help most?

## What Could Be Improved

- Where did you get stuck?
- What took longer than expected?
- Where did requirements change mid-cycle?
- What edge cases surprised you?
- Where did the process feel ceremonial vs valuable?

## Action Items

Based on the above, what specific changes should we make?

- Process changes
- Documentation updates
- Tool improvements
- Knowledge to capture

## Key Learnings

What should we remember for future cycles?

### Patterns to Reuse
- Successful approaches
- Effective techniques
- Good architectural decisions

### Anti-Patterns to Avoid
- Approaches that failed
- Time wasters
- Common mistakes

### Decisions to Document
- Architectural choices made
- Trade-offs accepted
- Rationale for key decisions

## Retrospective Checklist

- [ ] Discussed what went well
- [ ] Identified improvements
- [ ] Created actionable items
- [ ] Captured learnings in learnings.md
- [ ] Updated documentation if needed
